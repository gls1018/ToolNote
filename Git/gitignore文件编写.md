### 1. 注释

- 以 `#` 开头的行是注释

```
# 这是一个注释
```



### 2. 匹配模式

- **目录匹配**：以 `/` 结尾表示匹配目录

```
bin/      # 忽略所有名为 bin 的目录
```



- **文件匹配**：直接写文件名或使用通配符

```
*.log     # 忽略所有 .log 文件
```

### 3. 通配符

- `*`：匹配任意多个字符（除了路径分隔符）
- `?`：匹配单个字符
- `[]`：匹配括号内的任意一个字符

```
*.tmp          # 所有 .tmp 文件
test?.txt      # test1.txt, testA.txt 等
[abc].txt      # a.txt, b.txt, c.txt
```



### 4. 路径匹配

- 以 `/` 开头：只匹配当前目录下的文件/文件夹

```
/local.txt     # 只忽略根目录下的 local.txt
```



- 不以 `/` 开头：匹配所有位置的同名文件/文件夹

```
temp/          # 忽略所有名为 temp 的目录
```



### 5. 否定规则

- 以 `!` 开头：不忽略匹配的文件

```
*.log          # 忽略所有 .log 文件
!important.log # 但不忽略 important.log
```



### 6. 目录通配

- `**/`：匹配任意层级的目录

```
**/node_modules/  # 忽略所有 node_modules 目录
```



### 7. 范围匹配

- 使用 `[]` 匹配字符范围

```
[0-9].txt      # 0.txt, 1.txt, ..., 9.txt
[a-z].txt      # 所有小写字母开头的 .txt 文件
```
